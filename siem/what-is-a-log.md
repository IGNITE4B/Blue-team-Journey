# What is a Log?

## Definition

A log is a recorded event generated by a system, application, device, or service.  
Logs help security teams understand what happened inside an environment by documenting activities, actions, errors, connections, and user behavior.

From a SOC perspective, logs are how systems communicate operational activity to analysts.

---

## Why Logs Matter

Logs are important because they provide visibility into what is happening across systems and networks.

SOC analysts use logs to:

- Detect suspicious behavior
- Investigate security incidents
- Track failed login attempts
- Identify unauthorized access
- Understand attacker activity
- Build investigation timelines

Without logs, security teams would have very limited visibility during investigations.

---

## Types of Logs

Different systems generate different types of logs depending on their role and function.

Examples include:

- Operating system logs
- Authentication logs
- Firewall logs
- Web server logs
- Application logs
- Network device logs
- Cloud service logs

Each log source provides different operational information that helps analysts understand events inside an environment.

---

## Example of a Security Event

A user attempts to log into a server multiple times with the wrong password.

The system records:

- Username
- Time of attempt
- Source IP address
- Failed authentication status

If the failed attempts continue, the SIEM may trigger an alert for possible brute-force activity.

This allows analysts to investigate whether the activity is malicious or simply a user mistake.

---

## How SIEM Uses Logs

A SIEM collects logs from multiple systems and centralizes them in one platform.

The SIEM then:

- Correlates events
- Detects suspicious patterns
- Generates alerts
- Helps analysts investigate incidents

For example:

- Firewall logs show network traffic
- Authentication logs show login activity
- Endpoint logs show device behavior

The SIEM combines this information to provide better visibility during investigations.

---

## Windows Event Logs

Windows systems generate Event Logs that record system and user activity.

These logs commonly include:

- User logins
- Failed authentication attempts
- Service activity
- System errors
- Process execution events

SOC analysts frequently review Windows Event Logs during investigations because they provide detailed visibility into endpoint behavior.

---

## Firewall Logs

Firewall logs record allowed and blocked network connections.

They help analysts identify:

- Suspicious IP addresses
- Blocked traffic
- Unusual outbound connections
- Port scanning activity

Firewall logs are important for monitoring network security and identifying possible intrusion attempts.

---

## Authentication Logs

Authentication logs record login activity across systems and applications.

These logs may include:

- Successful logins
- Failed login attempts
- Account lockouts
- Privilege escalation events

SOC analysts use authentication logs to detect credential abuse, brute-force attacks, and unauthorized access attempts.

---

## My Understanding

I understand logs as operational evidence generated by systems during activity.

In a SOC environment, logs help analysts understand what users, devices, and applications are doing.  
They are essential for detecting threats, investigating incidents, and building visibility across an environment.

Without logs, a SOC analyst would have very limited ability to investigate suspicious activity.
